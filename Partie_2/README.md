# P13 - Partie 2 : Realiser un portfolio

Cette partie prepare la presentation portfolio du parcours Data Analyst.
L objectif est de transformer les livrables techniques en preuves claires, lisibles et actionnables pour un recruteur, un client ou un evaluateur.

---

## Objectif de la Partie 2

- Selectionner les projets les plus pertinents pour le poste cible.
- Structurer les preuves par competences (Python, SQL, BI, communication).
- Clarifier les resultats metier et la valeur decisionnelle.
- Publier une vitrine GitHub propre, avec liens publics verifies.

## Mission 2 - Realisation du portfolio

La Mission 2 formalise la narration des preuves de competences dans le portfolio.

Chaque preuve doit suivre le fil : **contexte -> besoin metier -> demarche -> resultats -> impact**.

Les resultats de veille sont integres dans les fiches et syntheses pour justifier les choix techniques : outils IA, methodes d'analyse, support de publication, niveau de documentation et limites.

Document detaille : `Réaliser un portfolio/docs/06_mission_2_realisation_portfolio.md`.

---

## Positionnement avec le portfolio prive

Le depot portfolio complet est maintenu en prive pour proteger :

- brouillons et notes de travail,
- donnees intermediaires,
- workflows internes et contenus non finalises.

Dans P13, on publie uniquement la vitrine utile a la soutenance :

- preuves publiques (README, captures, livrables),
- liens vers dashboards publics,
- liens vers depots projets publics.

Acces au portfolio prive : possible sur demande dans le cadre d un entretien ou d une soutenance.

## Plan de travail recommande

1. Completer la matrice portfolio (projets, competences, preuves, statuts).
2. Definir 3 a 5 projets phares a mettre en avant.
3. Harmoniser les README projets avec un meme gabarit.
4. Verifier les liens publics (depots, dashboards, livrables).
5. Finaliser un pitch court par projet (contexte, action, resultat, impact).

---

## Criteres de validation avant soutenance

- Le portfolio est lisible en moins de 5 minutes.
- Chaque projet phare affiche au moins 1 resultat chiffre.
- Les liens publics sont fonctionnels.
- Les fichiers sensibles ou lourds ne sont pas exposes.
- La coherence entre Partie 1 et Partie 2 est explicite.
