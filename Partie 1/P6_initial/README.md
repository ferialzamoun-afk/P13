# P6 initial - Bottleneck

## Role du dossier

Ce dossier conserve la version initiale du projet 6 Bottleneck, avant les ameliorations apportees dans le cadre du P13 Partie 1.

Il sert de reference pour comparer l'etat de depart avec la version amelioree par la demarche IA : cadrage, controles qualite data, documentation, reproductibilite et restitution.

## Contenu

| Dossier | Contenu | Role |
|---|---|---|
| `data/` | Fichiers `erp.xlsx`, `web.xlsx`, `liaison.xlsx` | Donnees sources du projet initial |
| `notebook P6/` | Notebook Bottleneck initial | Analyse initiale validee au projet 6 |
| `Extraits_P6/` | Presentation de validation | Preuve du rendu utilise pour valider le projet |

## Donnees disponibles

| Fichier | Description | Usage principal |
|---|---|---|
| `erp.xlsx` | Donnees issues de l'ERP | Prix, stocks, disponibilite, identifiants produit |
| `web.xlsx` | Donnees issues du site web | Ventes, informations produit web |
| `liaison.xlsx` | Table de correspondance | Rapprochement entre identifiants ERP et web |

## Statut

Cette version est conservee comme base initiale. Les modifications, controles complementaires, documents de cadrage et preuves IA doivent etre places dans le dossier `../P6_ameliore_IA/`.

## Utilisation dans P13 Partie 1

Le dossier permet de demontrer :

- l'etat initial du livrable P6 ;
- la provenance des donnees utilisees ;
- la comparaison avant/apres avec la version amelioree ;
- la continuite entre le projet valide et la mission P13.

## Points de vigilance

- Ne pas modifier les fichiers de ce dossier sans conserver une trace de changement.
- Ne pas ajouter ici les livrables P13 finaux pour garder une separation claire entre initial et ameliore.
- Documenter toute copie de fichier vers `P6_ameliore_IA` dans la synthese finale ou le plan projet.
